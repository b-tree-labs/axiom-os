# Getting help

Axiom is open source and **volunteer-maintained**. Help is best-effort and
offered by people doing this on their own time — please keep that in mind, and
you'll get a lot further.

## Where to ask

- **Questions, ideas, "how do I…?"** → [GitHub Discussions](https://github.com/b-tree-labs/axiom-os/discussions).
- **Bugs** → open an issue with the bug template (include `axi status`, repro
  steps, and logs with secrets redacted).
- **Feature ideas** → open an issue with the feature template, or start a
  discussion if it's exploratory.
- **Security issues** → privately, per [SECURITY.md](SECURITY.md) — not a public
  issue.

## What helps you get a good answer

- Search existing issues/discussions first.
- Share a **minimal reproduction**, your version, and what you already tried.
- Ask in public (issues/discussions), not in private DMs/email — so others
  benefit and any maintainer can pick it up.

## What to expect

- **No SLA.** Responses come when a maintainer has time. A polite nudge after a
  couple of weeks is fine.
- **"No" is a valid answer.** Not every request fits the project's scope or
  direction; we'll explain why (see [GOVERNANCE.md](GOVERNANCE.md)).
- **Commercial / dedicated support** is not offered through this repo. If your
  organization needs that, open a discussion to talk about options.
